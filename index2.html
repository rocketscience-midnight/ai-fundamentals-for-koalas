Lesson 1
Don't Ask One AI — Ask Three
Run the same prompt through multiple models and watch them disagree. That's where the insight lives.
⏱ 20 min

Why Different Models Think Differently

Every LLM was trained on different data, with different objectives and fine-tuning. ChatGPT tends to be thorough and tool-oriented. Claude tends to be crisp and structured, good at reasoning. Gemini is well-connected to Google's knowledge graph, useful for research. These aren't marketing claims — you'll actually see different outputs when you run the same prompt through each.

The point isn't to pick a winner. It's to collect perspectives. One model might surface a risk you hadn't considered. Another might challenge an assumption the first one made. When you synthesize their outputs yourself, the result is often better than any single response.

💡
Key Insight

You're the curator. The AI generates, you judge. That's the division of labor that works. Look for ideas that appear in all three (important), only in one (potentially most valuable), and contradictions (worth investigating).

The Council of LLMs — Starter Prompt

Write one clear, detailed prompt and run it through ChatGPT, Claude, and optionally Gemini. Don't tweak it between runs — you want comparability.

prompt template 
Act as an expert in [your domain].
I need to [describe task].

The constraints are:
- [constraint 1]
- [constraint 2]
- [constraint 3]

Also flag the things most people forget,
and highlight the biggest risks.

Assign Roles for Better Results

You can make the council even more useful by assigning different roles. The role changes behavior meaningfully — ask Claude to be a critic and it will find things it wouldn't surface if you just asked for ideas.

🎨 Generator

Brainstorm without filtering — quantity over quality

🔍 Critic

Find gaps and weaknesses in a plan you already have

😈 Devil's Advocate

Argue against your current plan — why will it fail?

🔬 Researcher

Look up known issues with similar situations (enable web search)

critic role 
Act as a critical reviewer.
Here is my current approach: [paste your approach].
Find the weaknesses, blind spots, and things I'm missing.
Be honest — I want to improve this, not hear that it's great.
devil's advocate 
Act as a devil's advocate.
I'm planning to [describe your plan].
Argue against it. Tell me why this will fail.
What am I not seeing?

One LLM, Many Interfaces

Even within a single LLM, the interface shapes what you get. Take Claude: the Web with Connectors version can access your email and Google Drive. The Desktop App reads local files directly without uploading. Claude Code (CLI) is built for technical work and behaves more like an engineer. The Chrome Extension analyzes whatever page you have open.

Same underlying model, four meaningfully different experiences. Match the tool to the job.

🔒
Privacy Note

Don't paste real customer data, internal code, or anything confidential into public AI tools. Anonymize or use synthetic examples. When in doubt, ask.

✍️ Your Exercise — ~20 minutes

Pick a task you're currently working on. Do both parts:

  • ✓
    Part 1: Write one clear prompt and run the exact same prompt in both ChatGPT and Claude. What does one surface that the other doesn't? Where do they contradict?
  • ✓
    Part 2: Take one LLM and run your task twice — first with your original prompt, then with the Critic role prompt. Compare the outputs.
  • ✓
    Try LM Arena (lmarena.ai) to compare two LLMs side by side without switching tabs — free and cuts the friction significantly.

Lesson 2
Teaching AI to Remember You
Stop re-explaining yourself every chat. 15 minutes of setup, endless hours saved.
⏱ 15 min

The Right Way: Set It Explicitly

LLMs don't have a persistent brain between sessions by default — each conversation starts blank. Memory features let information carry over. The distinction between implicit (AI infers patterns from history) and explicit (you tell it what to remember) matters: explicit is always more reliable.

memory setup prompt 
Remember this about me:
I work as a software tester.
My tech stack is C#, Selenium 4, and .NET 8.
I prefer bullet-pointed outputs for test cases,
and I want output formatted so it can be pasted
directly into our test management tool.

After this, open a new chat. You shouldn't need to re-explain any of that. In Gemini, there's a dedicated "Saved Info" section where you can view and edit what the AI has stored.

What to Actually Store

Role & domain — "I'm a QA lead testing a B2B SaaS product"

Tech stack — "We use Playwright with TypeScript, deployed on AWS"

Team terminology — "We call end-to-end tests 'journey tests'"

Output format — tables, bullets, numbered steps, detail level

Current project — "Testing the new user onboarding flow for v4.2"

Documentation style — technical or business-readable?

Tone of Voice & Output Format

AI doesn't automatically adapt to your voice — it defaults to "confident, structured, slightly corporate." Once you tell it what you want, ChatGPT and Claude both follow style direction well.

style + format prompt 
Write a summary of these test results for a team standup.
Audience: developers and a product manager, no testing jargon.
Tone: direct, brief, no filler.
Format: 3 bullet points max. Each bullet is one sentence.
Don't start with "Here are" or "This summary".

Test results to summarize:
[paste your test results here]
🧠
Memory vs RAG

Memory is like RAM — fast, limited, used in every session. RAG (custom assistants with uploaded docs from Lesson 4) is like a hard drive — larger capacity, looked up on demand. Both have their place and work well together.

🗓️
Maintenance Tip

Memory isn't forever. Projects end, stacks change, old context becomes noise. Review your stored memory every few months and clean out anything stale.

✍️ Your Exercise — ~15 minutes

Do this now. Every AI session from this point on will start with a head start.

  • ✓
    Open ChatGPT or Claude and set your context explicitly: role, tech stack, output preferences. Keep it to 3–5 sentences.
  • ✓
    Start a new conversation and ask something testing-related without providing any background. See what it already knows.
  • ✓
    Review stored memory: ChatGPT → Settings → Personalization → Memory. Claude → Settings → Activity. Edit or remove anything outdated.

Lesson 3
Stop Describing Screenshots — Just Send Them
AI can see. A screenshot conveys more than 2000 tokens of text description. Start using this.
⏱ 15 min

A Picture Is Worth More Than You Think

When you upload a screenshot to an AI, you're giving it roughly 2,000 to 5,000 tokens of information in a format that often conveys more than the equivalent amount of text. Writing about a UI bug in words is harder, slower, and less accurate than just screenshotting it and asking the AI what's wrong.

visual review prompt 
Act as a testing expert.
Review this screenshot and report any accessibility,
usability, and UI issues you notice.
Be specific.
Mention which elements are affected and why it's a problem.

What Images Are Good For

Issue detection — accessibility, visual regressions, anything hard to articulate

Comparing states — before/after a deploy, ask AI to describe differences

Design validation — wireframe vs. implementation: does it match?

Bug report writing — upload the bug, get a consistent, complete description

Voice: Underused and Actually Useful

Typing is slower than speaking. For complex testing scenarios — edge cases, long sequences of steps, convoluted requirements — consider voice input instead. ChatGPT's Advanced Voice Mode lets you have a real-time conversation. Claude has voice input in its mobile app.

🌐
Pro Tip

If you're a non-native English speaker dictating in English but your phone is set to German, transcription quality suffers. Switch your device language to match the language you're speaking — the difference can be significant.

Screen Sharing and Live Analysis

ChatGPT's Advanced Voice Mode can analyze what's on your screen while you talk to it. Claude's mobile app supports visual input from your camera. For quick walkthroughs of a content-heavy page, the ability to share your screen and ask "does this make sense?" can save meaningful time.

⚠️
Limitation

AI vision isn't infallible. It can miss things, misread text, or misinterpret visual hierarchy. Treat AI-generated issue lists as a first pass to augment your own review — not a replacement for it.

✍️ Your Exercise — do at least one
  • ✓
    Next time you find a visual bug, take a screenshot and paste it into Claude or ChatGPT: "Act as a tester. What issues do you see in this screenshot?" Compare what it finds to what you noticed.
  • ✓
    Try voice input for your next complex prompt. Use ChatGPT's voice mode or just dictate via your phone. Is it faster? If quality is poor, try switching device language.
  • ✓
    Upload a screenshot of a UI you've been testing and ask specifically for an accessibility review. You might be surprised what it flags.

Lesson 4
Build an AI That Actually Knows Your Job
Build it once, use it forever. Custom assistants eliminate the prompting tax from every conversation.
⏱ 20 min

The Problem with Generic AI

Every AI tool was built for everyone. That's its strength, and its limitation. It doesn't know your team's definition of "done," your bug report template, or the quirks of the system you test. Custom assistants move that work out of every conversation and into a one-time setup.

The Three Ingredients

Instructions — the role, behavior, output format, what it should and shouldn't do. Quality here determines quality everywhere.

Knowledge base — upload PDFs, guidelines, past test plans, acceptance criteria templates. AI draws on these when answering.

Capabilities — web search, code execution, image analysis: turn on/off. For testing assistants: web search off, image analysis on.

Where to Build

🤖 ChatGPT

"Create a GPT" in the sidebar. Large marketplace to browse for inspiration.

🟠 Claude

"Projects" in the sidebar. Custom instructions + knowledge files scoped to a use case.

💎 Gemini

"Gems" at gemini.google.com. Similar concept, Google's ecosystem.

🔬 AI Studio

System Instructions + saved configurations. Good for iterating before committing.

Requirements Analyzer — Complete Instructions

Here's a complete, ready-to-use example. Paste this directly as the system instructions for your assistant:

custom assistant instructions 
Act as an expert software tester who regularly
reviews feature requirements for testability.

When given a requirement, provide the following
in this order:
1. Clarification questions: things you'd need
   answered before you could test this properly
2. Key notes: important implications a tester
   should be aware of
3. Risks: what could go wrong, what's likely to break
4. Test ideas: concrete scenarios worth testing
5. Requirement gaps: anything that's missing,
   ambiguous, or contradictory

Be specific. Don't pad the output with generic advice.
If something in the requirement is unclear, say so directly.

Wait for the user to provide a requirement before responding.

Notice the last line: "Wait for the user to provide a requirement before responding." Without it, the AI might try to be helpful by generating a generic example. That instruction keeps it focused.

Other Assistants Worth Building

Bug Report Writer — describe what happened, get a formatted bug report in your team's template

Test Data Generator — specify what you need, get structured edge-case data back

Root Cause Helper — describe a pattern of failures, get hypotheses about causes

Standup Summarizer — paste test results, get a stakeholder-ready summary

🔗
Team Multiplier

Custom assistants are a bridge between people who know how to prompt and people who don't. Once you've built something that works well, sharing it with the team means everyone benefits — not just the person who figured out the right prompt.

🔒
Privacy

Don't upload anything you wouldn't want stored on an external platform. Internal architecture docs, customer data, proprietary test plans — use anonymized or generalized versions when building knowledge bases.

✍️ Your Exercise — build something concrete
  • ✓
    Identify one repetitive task in your daily testing work — something you do at least weekly with a consistent pattern. That's your candidate.
  • ✓
    Browse the ChatGPT GPT Store and search for "software testing" or "QA". Look at what others have built for ideas and instruction patterns.
  • ✓
    Build a Requirements Analyzer using the instruction template above. Test it on a real requirement from your current sprint. Note what needs refinement.

Lesson 5
AI Hallucinations & The Power of Good Context
AI doesn't lie — it pattern-matches. Understanding this lets you prevent hallucinations almost entirely.
⏱ 20 min

What's Actually Happening

LLMs don't retrieve facts from a database. They generate text by predicting the next word based on patterns in training data. When the model doesn't have enough reliable training data for a specific question, it fills the gap with something plausible-sounding — whether or not it's true. The confidence doesn't correlate with accuracy.

Hallucinations are more likely with:

Vague or underspecified prompts — the less context, the more the AI fills in

Questions about recent events or very specific domain knowledge

Edge cases that weren't well-represented in training data

Prompts that implicitly reward confident-sounding output over accurate output

Chain of Verification (CoVe)

The most effective technique for reducing hallucinations in testing contexts. Instead of just producing output, ask the AI to produce output and then check it against specific criteria.

without verification ❌ 
Generate test cases for the login functionality.
with Chain of Verification ✅ 
Act as a software tester.
Design test cases for the login functionality
using chain of verification:
1. For each test case, state what you're testing
   and why it matters
2. Verify whether each test is realistic and
   actually needed
3. Verify whether each test is valid for this
   specific context:
   [describe your actual system]
Flag any assumptions you're making.

The AI Maturity Staircase

  • 1
    Basic Chat
    Type questions, hope for something useful. Inconsistent quality.
  • 2
    Prompt Engineering
    Structured, deliberate requests. Better questions, better answers.
  • 3
    Context Engineering
    Deliberately providing the right information and constraints. Biggest gains are here.
  • 4
    Agentic Engineering
    AI uses tools and takes real actions. Covered in Lesson 8.

Teaching the AI to Say "I Don't Know"

uncertainty instruction 
If you don't have enough information to answer
accurately, say so.
Do not guess or extrapolate.
If the requirements are incomplete, respond with:
"Requirements incomplete. Please clarify
[specific thing] before I can generate
accurate test cases."
🌡️
Temperature Setting

For testing work (test cases, requirements analysis, bug reports) — use lower temperature (0.3 or below). For brainstorming — 0.7–1.0 gives more variation. Lower temperature = fewer hallucinations. You can adjust this in Google AI Studio (Lesson 7).

💡
Mental Model

Think of hallucinations as a signal about prompt quality, not AI unreliability. When you get a hallucinated response, ask: "What did my prompt fail to specify?" The fix is almost always more context or more structure.

✍️ Your Exercise — 10–15 minutes each
  • ✓
    Take a prompt you've used before that generated questionable output. Rewrite it with CoVe: ask the AI to verify each point against your actual context.
  • ✓
    Add the "I don't know" instruction to your next complex prompt. Notice whether the output changes — does the AI flag more uncertainty?
  • ✓
    Next time you get test cases from AI, pick two or three and verify them manually. Calibrate your own sense of when AI output is reliable and when it needs more scrutiny.

Lesson 6
Your Feedback Shapes AI — Here's How to Use That
Those thumbs up/down buttons are training levers. Learn to use them to calibrate AI to your standards.
⏱ 15 min

What RLHF Actually Is

Reinforcement Learning from Human Feedback. After an LLM generates a response, humans evaluate it. Those evaluations update the model — rewarding patterns that got positive feedback, penalizing patterns that got negative feedback. This is how ChatGPT, Claude, and Gemini were all trained and continue to be refined.

Your individual click won't reshape the model overnight, but within your own usage history, the feedback you give shapes how that tool responds to you specifically.

When Thumbs Down Gets Interesting

Most people click thumbs down when annoyed and move on. The more useful approach: actually tell the AI what was wrong. In ChatGPT, it asks for specifics when you click thumbs down:

"Factually incorrect" — helps the model learn accuracy

"Didn't follow instructions" — helps it learn to be more literal

"Style wrong" — calibrates tone and format to your preferences

"Memory issue" — something from previous sessions is bleeding in wrong

Where Testers Have an Unusual Advantage

Most software tasks are fairly objective — code works or it doesn't, a test passes or fails. But a significant part of testing involves judgment calls that aren't binary: Is this UI intuitive enough? Is this error message clear? Is this documentation at the right level?

RLHF is particularly well-suited for training an AI on exactly these kinds of subjective preferences:

📝 Bug Reports

Wrong structure? Thumbs down + note the correct order. Format adapts over time.

📖 Documentation

Technical vs. business-readable? Consistent feedback calibrates which default you get.

📋 Test Cases

If explanations of changes help your team, upvote that pattern. It surfaces more often.

📊 Summaries

Too verbose? Too terse? Consistent feedback over weeks makes a measurable difference.

📈
The Compound Effect

You don't need elaborate feedback on every interaction. The habit that makes a difference is being consistent: when you get output you really like, click thumbs up. When something is clearly off, click thumbs down and add a one-line note. Done consistently over weeks, this calibrates your AI tools to your preferences in ways that save time every day.

🎯
Objective vs. Subjective

Objective facts — whether a test case is technically valid, whether code compiles — aren't improved by feedback in the same way. Fix those with better prompting and more context (Lesson 5). Feedback shines anywhere output is evaluated against preferences and judgment.

✍️ Your Exercise — start the habit this week
  • ✓
    Next time you use ChatGPT or Claude and get a response formatted exactly how you like it — click thumbs up. It takes one second. Build the habit.
  • ✓
    If you get a response that's too verbose, vague, or wrong format — click thumbs down and specify which of those it was. Notice whether subsequent responses improve.
  • ✓
    Identify one subjective quality standard your team cares about: documentation clarity, bug report format, or test case structure. Use it as a feedback filter for two weeks.

Bonus Lesson 7 ⭐ Bonus
Google AI Studio — A Power Tool for Serious Prompting
The free power tool most teams ignore. Compare models, build apps, see the controls nobody else touches.
⏱ 30 min

Why the Chat Interface Isn't Enough

Standard chat is great for quick tasks. But if you're trying to develop a prompt you'll use repeatedly, or understand why the same prompt produces wildly different results sometimes, a chat window doesn't give you the tools to investigate or iterate systematically.

Think of it like the difference between writing code in Notepad versus in an IDE. Google AI Studio is the IDE — free, full-featured, with controls that even paid tools don't always offer.

Compare Mode — The Feature That Changes Everything

Run the same prompt through two different models (or the same model with different settings) side by side. Want to know whether higher or lower temperature produces better test cases? Run them side by side. Want to compare Gemini 2.5 Pro vs. Flash? You get both outputs simultaneously without copying between tabs.

⭐
Access

Google AI Studio is free at aistudio.google.com. No subscription required. Just a Google account. The paid tier (Gemini API) is needed for high-volume use — for exploration and team tooling, the free tier is plenty.

System Instructions: Define the AI's Persona

On the right side of the interface, set System Instructions that frame every response in a conversation — the AI's role, tone, and behavioral constraints without repeating them in every message. It's a great place to experiment with instructions before committing them to a permanent custom assistant.

system instructions example 
You are an expert software tester specializing
in requirements analysis.
When given a requirement, always identify
testability issues before suggesting test cases.
Use clear, numbered lists for test cases.
Keep responses concise. No padding.

Visible Controls: Temperature & Token Count

Temperature slider — visible and adjustable. 0.3 for precise work, 0.7–1.0 for brainstorming.

Token count — real-time display of your prompt's size. Helps you stay within context limits and understand prompt cost.

Model selection — switch between Gemini versions, Flash, Pro, and experimental models instantly.

Build Apps — Prompts That Do Something

The most underrated feature: "Build Apps" converts your prompt into a working web application anyone can use. Write a prompt describing a test data generator, and AI Studio builds an interactive app with a form where you input parameters. You've turned a prompt into a team tool — in minutes.

test data generator app prompt 
Act as a software testing expert.
Create a test data generator for user registration forms.
Let the user specify:
- Number of records
- Types of edge cases to include
  (invalid emails, special characters, boundary lengths)
- Output format (CSV or JSON)

Generate realistic, varied test data that covers
both happy path and edge cases.

The Same Ideas, Three Playgrounds

🔬 Google AI Studio

Free, full-featured, unique compare mode. Best starting point.

🟠 Anthropic Console

console.anthropic.com — for Claude. Workbench for iterating prompts with full control.

🤖 OpenAI Playground

platform.openai.com/playground — for GPT. System messages, temperature, model selection.

✍️ Your Exercise — ~30 minutes
  • ✓
    Go to aistudio.google.com and run a prompt you use regularly. Then enable Compare mode with a different temperature setting or model. Look for meaningful differences.
  • ✓
    Try the Build Apps feature with a simple testing task: bug report formatter, test case template generator, or checklist builder. Share the app link with one other person.
  • ✓
    Set up System Instructions with your testing context. Save the session. Then try the same System Instructions in the Anthropic Console Workbench — compare how the two models respond.

Bonus Lesson 8 ⭐ Bonus
From Chat to Coworker
AI that doesn't just answer — it acts. Browses the web, runs code, takes real steps. Here's what's actually real today.
⏱ 20 min

What Is Agentic AI?

Regular AI works like a very well-read colleague who can only communicate in writing. You ask, it responds. The interaction starts and ends there.

Agentic AI is different. Instead of just answering, it can take actions: browse a live website and read what's actually on it today, run a piece of code and report what happened, call an external API and process the response, chain multiple steps together — plan, act, observe, adjust, continue.

🤔
The Key Difference

Asking "what does the current OWASP Top 10 look like?" vs. asking an agent to go look it up, compare it to last year's version, and come back with a clear summary of what changed. The second is a task with real steps that require interacting with the world.

What You Already Have Access To

ChatGPT Pro — web browsing opens a real browser and reads live pages. Code interpreter executes actual Python. These are real actions, not simulations.

Claude with tool use — can browse the web, run code, interact with external systems in the right configurations.

Claude Code (CLI) — reads and writes files on your local machine, runs terminal commands, works through multi-step tasks with a degree of autonomy that feels genuinely different.

Google AI Studio — function calling lets you define what capabilities the AI can use, then let it decide when to invoke them.

The Building Blocks

🔧 Tool Use

AI calls externalcapabilities — web search, code runner, file system — and incorporates results into its response.

📡 Function Calling

You define what functions exist ("get_ticket_status") and the model invokes them when relevant. Foundation for wiring AI to real systems.

🔗 Multi-step Reasoning

Plan a sequence of actions, execute the first step, look at what happened, decide what to do next — until the task is done.

🔌 MCP

Model Context Protocol — an emerging standard for how AI models connect to external tools and data sources consistently.

Reality Check: What Works Today vs. What's Hype

Web browsing in ChatGPT/Claude ✅ Works well, useful today. Great for current security advisories, live docs, competitor features.

Code execution in ChatGPT ✅ Solid. Great for generating and immediately running scripts, analyzing files, doing calculations.

Claude Code ✅ Powerful for developer workflows. Requires comfort with the terminal. Worth it if you're in that environment.

Fully autonomous AI testing agents ⚠️ Still early. Demos look impressive. Reliability in real, messy environments is inconsistent. Watch, don't deploy in production yet.

🔌
MCP — Worth Knowing

If you hear people talking about "MCP servers" or "AI integrations" in the coming months — this is what they mean. Model Context Protocol is an emerging standard that defines how AI models connect to external tools and data sources in a consistent way. You don't need the technical details yet, just the concept.

option A — web search 
Search the web for the current OWASP Top 10 and give me
a clear summary of what changed compared to the previous version.
Be specific about what moved up the list, what dropped off,
and what's newly included.
option B — code execution 
Write a Python script that generates 50 realistic test users,
each with a name, email address, age between 18 and 65,
and a country selected from a realistic distribution.
Then run it and show me the first 10 results.
🔒
Heads Up

When using AI tools that browse the web or run code, be thoughtful about what you're asking them to access or generate. Agentic tools have more reach than a standard chat — which is exactly why they're useful, and why awareness matters.

✍️ Your Exercise — notice the difference
  • ✓
    Try Option A (OWASP web search) in ChatGPT or Claude with web search enabled. Watch whether it actually opens a browser and reads live pages.
  • ✓
    Try Option B (test user generation) with code execution enabled. Compare that experience to asking the same question in plain chat without tools.
  • ✓
    Think about one workflow in your team that could benefit from agentic AI: querying your staging API, cross-referencing test reports with Jira tickets, or executing test steps. Write down what it would look like.
