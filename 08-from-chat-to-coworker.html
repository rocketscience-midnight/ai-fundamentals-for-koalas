⭐ Bonus Lesson

#8: From Chat to Coworker

Up until now, we've mostly talked about AI as something you ask questions and it answers. That model still works and you should keep using it. But there's a shift happening in how AI is actually being deployed, and parts of it are already running in the tools sitting on your desk right now.

What Is Agentic AI?

Regular AI works roughly like a very well-read colleague who can only communicate in writing. You ask, it responds. The interaction starts and ends there. It works entirely from what it knows.

Agentic AI is different. Instead of just answering, it can take actions. Browse a live website and read what's actually on it today. Run a piece of code and report what happened. Call an external API and process the response. Chain multiple steps together — plan, act, observe, adjust, continue — all within a single task.

Think of the difference between asking a colleague "what does the current OWASP Top 10 look like?" versus asking them to go look it up, compare it to last year's version, and come back with a clear summary of what changed. The first is a question. The second is a task with real steps that require interacting with the world. Agentic AI can do the second kind.

What You Already Have Access To

ChatGPT Pro includes web browsing and a code interpreter. When you ask it to look something up, it actually opens a browser and reads real pages — not cached training data from two years ago, but live content. When you ask it to analyze a spreadsheet or write and run a script, it executes actual Python code and shows you the real output. These aren't simulations. They're real actions.

Claude with tool use can browse the web, run code, and in the right configurations interact with external systems. Claude Code — the CLI version — can read and write files on your local machine, run terminal commands, and work through multi-step development tasks with a degree of autonomy that feels meaningfully different from a chat interface.

Google AI Studio supports function calling — a structured way to define what capabilities the AI can use, and let it decide when to invoke them based on what you ask. This is how you'd start building AI that connects to real data sources.

The Building Blocks

Tool use is the ability for an AI model to call external capabilities — a web search engine, a code runner, a file system, a database. The model decides when a tool is needed based on your request, uses it, gets the result back, and incorporates that into its response. From your side, it often just looks like a smarter, more capable answer.

Function calling is the more structured version. You or a developer defines what functions exist — "get_ticket_status", "query_test_database", "check_api_endpoint" — and the model can invoke them when relevant. This is the foundation for wiring AI to your actual systems, rather than the general-purpose ones that come built in.

Multi-step reasoning ties it all together. An agentic AI doesn't just respond once. It can plan a sequence of actions, execute the first step, look at what happened, decide what to do next, and keep going until the task is done. Or until something goes wrong, which is why human oversight still matters a lot.

Why This Matters for Testers Specifically

Agentic AI opens up the possibility of the AI actually interacting with the thing you're testing. An AI that can query your staging API directly and tell you whether the response schema matches the spec. One that can parse your test report, cross-reference it with open Jira tickets, and flag anything that looks inconsistent. One that can take a set of test steps and actually execute them in a browser while logging what it observes.

That's not all available out of the box today — but the foundation is being built now. Knowing what's coming helps you recognize it when it arrives, and ask smarter questions about how your team could use it.

🔌 One infrastructure term worth knowing: MCP, or Model Context Protocol. It's an emerging standard that defines how AI models connect to external tools and data sources in a consistent way. If you hear people talking about "MCP servers" or "AI integrations" in the coming months, this is what they mean. You don't need the technical details, just the concept.

Reality Check: What Works Today vs. What's Hype

Web browsing in ChatGPT and Claude? Works well, useful today, use it now. It's especially good for anything where you need current information — recent security advisories, updated documentation, what a competitor's feature actually does.

Code execution in ChatGPT? Solid. Great for generating and immediately running scripts, analyzing files, doing calculations you don't want to do manually.

Claude Code? Powerful for developer workflows. Requires some comfort with the terminal to set up and use well. Worth exploring if you're comfortable in that environment.

Fully autonomous AI testing agents that can plan, execute, and verify test cases end-to-end without supervision? Still early. The demos look impressive. Reliability in real, messy environments is inconsistent. This is a space to watch, not deploy in production workflows yet.

A Stepping Stone You've Already Taken

If you experimented with building a custom assistant in article #4, you already touched the edge of this. A custom assistant with specific instructions, a defined persona, and knowledge of your project is a small step in the agentic direction. You've defined what the AI knows and how it should behave. The next step is giving that AI tools — the ability to act on what you've asked, not just respond to it.

Your Exercise

Open ChatGPT or Claude — whichever has web search or code execution enabled for your account. Give it a task that actually requires using a tool, not just recalling information from training.

Option A (web search): "Search the web for the current OWASP Top 10 and give me a clear summary of what changed compared to the previous version. Be specific about what moved up the list, what dropped off, and what's newly included."

Option B (code execution): "Write a Python script that generates 50 realistic test users, each with a name, email address, age between 18 and 65, and a country selected from a realistic distribution. Then run it and show me the first 10 results."

Watch what happens. Notice whether the model opens a browser, runs actual code, or retrieves real-time data. Compare that experience to asking the same question in a plain chat where no tools are available.

🔒 Remember: when using AI tools that browse the web or run code, be thoughtful about what you're asking them to access or generate. Agentic tools have more reach than a standard chat — which is exactly why they're useful, and why awareness matters.
