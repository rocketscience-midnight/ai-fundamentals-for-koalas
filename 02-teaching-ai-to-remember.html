Every time you start a new chat with an AI, it has no idea who you are, what you test, or how you like your output formatted. You end up re-explaining the same context over and over. Memory features exist to fix exactly this, and once you set them up properly, they genuinely save time.

What AI Memory Actually Means

LLMs don't have a persistent brain between sessions by default. Each conversation starts blank. Memory features are the mechanism that lets information carry over, either automatically (implicit) or because you explicitly told the AI to remember something.

The distinction matters. Implicit memory tries to infer patterns from your past conversations. It can be useful, but it can also pick up things you didn't intend, and occasionally lead to the AI making assumptions based on outdated context. Explicit memory, where you deliberately tell the AI what to remember, is more reliable. More on that in a moment.

Long-term memory (memory that persists across sessions, not just within one conversation) is generally a paid feature. With Claude Pro and ChatGPT Pro, you have access to it.

The Right Way: Set It Explicitly

The fastest way to get value from memory is to tell the AI exactly what you want it to remember. In ChatGPT, you can do this right in the chat:


Remember this about me: I work as a software tester. My tech stack is C#, Selenium 4, and .NET 8. I prefer bullet-pointed outputs for test cases, and I want output formatted so it can be pasted directly into our test management tool.

After this, open a new chat. You shouldn't need to re-explain any of that. The AI will use that context automatically.

In Gemini, there's a dedicated "Saved Info" section where you can view and edit what the AI has stored. It's the closest thing to a transparency window into what the AI thinks it knows about you. For example: set "I am working on testing the Ministry of Testing website", then refine it to "I am testing the login feature". Open a new chat and ask for test scenarios with personas, and Gemini will use that stored context without you repeating it.

What to Actually Store

Think of memory in three buckets:

Facts about your context

These are the basics the AI needs to give you relevant answers without you repeating yourself:

  • Your role and domain ("I'm a QA lead, primarily testing a B2B SaaS product")
  • Your tech stack ("We use Playwright with TypeScript, deployed on AWS")
  • Your team's terminology ("We call the end-to-end tests 'journey tests', not 'E2E tests'")
  • Any acronyms or internal names the AI might not know

    • Your preferences for output

    This is where memory really pays off. Once the AI knows how you like things, you stop fighting the formatting:

  • Output format: tables, bullets, or numbered steps - whatever works best for you
  • Detail level: do you want the full picture or a concise summary?
  • Documentation style: technical or business-readable?

    • Current projects

    If you're deep in a specific feature or release cycle, storing that context means you can jump straight to questions without the preamble. "I'm currently testing the new user onboarding flow for v4.2" is worth storing when it's relevant.

    💡 Memory isn't forever. Projects end, stacks change, and old context can become noise. Build in a habit of reviewing your stored memory every few months and cleaning out anything stale.

    Memory is also distinct from RAG (Retrieval-Augmented Generation), which is what custom assistants use when you upload knowledge files (covered in article #4). Think of memory like RAM: fast, limited in size, and used for context you need in every session. RAG is more like a hard drive: larger capacity, better suited for documents and reference material you look up on demand. Both have their place, and they work well together.

    Implicit Memory: Use With Caution

    Both ChatGPT and Claude can pick up patterns from your conversation history without you explicitly asking them to remember anything. This can be convenient, but it can also quietly accumulate incorrect assumptions. If an AI ever seems to be drawing on context you didn't intend, check the memory settings and clean up what's there.

    The general principle: explicit is better than implicit. A few minutes setting up clear memory beats hoping the AI inferred the right things.

    Your Exercise

    This takes about 15 minutes. Do it now, and every AI session you run from this point on will start with a head start.

  • Open ChatGPT or Claude and set your context explicitly: role, tech stack, and output preferences. Keep it to 3-5 sentences.
  • Start a new conversation and ask something testing-related without providing any background. See what it already knows.
  • If you use ChatGPT, go to Settings → Personalization → Memory and review what's stored. In Claude, check Settings → Activity to see what it remembers. Edit or remove anything that's outdated or wrong.
